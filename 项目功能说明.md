# 🍽️ 今天吃什么 — 微信小程序项目功能说明

> **项目名称**: 今天吃什么 (eat-what)
> **AppID**: `wx59d58721596c93b1`
> **技术栈**: 微信小程序原生开发（无框架）+ 静态 JSON 菜谱数据 + 规则推荐引擎 + 讯飞星火 AI fallback
> **云开发环境**: `cloudbase-d0ggsef0eee4e6534`（投票系统已改为纯本地存储）
> **文档更新日期**: 2026-06-04

---

## 📋 项目概述

"今天吃什么"是一款帮助用户解决"每天吃什么"难题的微信小程序。核心功能包括：智能菜谱推荐、冰箱食材匹配推荐、减脂餐专区、多人投票选餐、随机转盘决策，以及个人口味偏好管理和菜谱收藏。

### 1. 🏠 首页（pages/home/home）

**路径**: `/pages/home/home`
**类型**: tabBar 页面（第一个 tab，应用启动页）

首页是应用的主入口和核心导航枢纽，采用 **Bento Grid** 风格的模块化布局。页面根据当前时间段展示个性化问候语，提供四个快捷功能入口（冰箱推荐、减脂餐、发起投票、随机转盘），并展示今日热门菜谱推荐。

#### 核心逻辑

- **搜索入口**: 点击搜索框跳转至菜谱列表页的搜索模式
- **快捷导航**: 冰箱推荐→冰箱页、减脂餐→减脂餐页、发起投票→投票房间页、随机转盘→转盘页，自动判断 tabBar/非tabBar 使用对应的跳转方式

#### 核心逻辑

- **食材分类**: 6 大分类共 50+ 种食材 — 蔬菜(13种)、肉类(7种)、海鲜(4种)、蛋奶(4种)、主食(5种)、调料(20种)
- **食材选择**: 点击选中/取消，已选食材显示在顶部栏，支持点击移除和一键清空
- **食材搜索**: 输入关键词从全部食材中模糊匹配
- **持久化**: 已选食材保存到 localStorage (`eat_what_fridge_selected`)，下次进入自动恢复

#### 推荐引擎（双引擎策略）

1. **静态匹配引擎** (`matchByIngredients`)：
   - 计算用户已选食材与每道菜谱的匹配率
   - 匹配率 ≥ 30% 的菜谱进入候选列表
   - 返回匹配率百分比和缺少的食材列表

2. **AI 补充引擎** (`aiRecommendByIngredients`)：
   - 当静态匹配结果不足 2 个时自动触发
   - 调用讯飞星火 AI API（Anthropic 兼容接口）
   - 发送食材列表，获取 JSON 格式的菜谱推荐
   - 与静态结果合并去重

#### 购物清单功能

- 列出每道推荐菜谱缺少的食材
- 支持一键复制到剪贴板
- 弹窗展示，方便用户去超市采购

#### 推荐结果卡片信息

每张结果卡片包含：菜谱图标、菜名、烹饪时间、卡路里、收藏按钮、匹配率标记、用料标签（已有的标✓）、缺少食材提示、可展开的烹饪步骤、视频教程链接

#### 功能描述

投票大厅是多人协作选餐功能的入口页面。用户可以发起投票让大家一起决定吃什么，也可以通过投票码加入他人创建的投票。页面还展示历史投票记录，并自动检测剪贴板中的投票码。

#### 页面结构

```
┌──────────────────────────────────┐
│  ┌──────────────┬──────────────┐ │
│  │   ➕ 发起投票  │   🔗 加入投票 │ │

│  │   投票房间    │   加入房间   │ │
│  └──────────────┴──────────────┘ │
├──────────────────────────────────┤
│  📋 我的投票                     │
│  ┌────────────────────────────┐  │
│  │ 🗳️ 今天中午吃什么          │  │
│  │    房间码: A3K7M2           │  │
│  │    状态: 进行中 🟢          │  │
│  ├────────────────────────────┤  │
│  │ 🗳️ 周末聚餐选哪家          │  │
│  │    房间码: B5N8P4           │  │
│  │    状态: 已结束 ⚫          │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

#### 核心逻辑

- **创建投票**: 点击"发起投票"跳转到投票房间页（创建模式）
- **加入投票**: 弹出输入框让用户粘贴投票码，解码后跳转到投票房间
- **剪贴板检测**: 每次进入页面自动读取剪贴板，检测是否包含投票码格式（`投票码:XXXXXX|encoded_data` 或 `投票码:XXXXXX`），发现后弹窗提示是否加入
- **历史记录**: 展示所有参与过的投票，点击可重新进入查看结果
- **投票码格式**: 6 位大写字母+数字（排除易混淆字符 I/L/O/0/1）

- **收藏管理**: 可展开/收起收藏列表，每项显示菜名、时间、卡路里，支持删除（带确认弹窗）和展开查看烹饪步骤
- **数据持久化**: 偏好设置保存到 localStorage (`user_profile`)，收藏保存到 `eat_what_favorites`

### 9. 🎰 随机转盘（pages/index/index）

**路径**: `/pages/index/index`
**类型**: 普通页面（从首页快捷入口进入）

#### 功能描述

随机转盘是一个趣味选餐工具，帮助用户在犹豫不决时随机选择吃什么。支持两种随机方式：**Canvas 转盘旋转**（视觉效果丰富）和**老虎机轮播**（快速简洁）。用户还可以管理自定义食物列表，添加自己喜欢但预设中没有的菜品。

#### 页面结构

```
┌──────────────────────────────────┐
│           🍽️ 结果区              │
│            🍜                    │
│         红烧牛肉面                │
│          [中餐]                  │
├──────────────────────────────────┤
│              ▼ 指针              │
│  ┌──────────────────────────┐    │
│  │                          │    │
│  │      🎨 Canvas 转盘       │    │
│  │     (彩色扇区+食物名)     │    │
│  │                          │    │
│  └──────────────────────────┘    │
│                                  │
│     [🎰 转一转]    [🎲 随机]     │
├──────────────────────────────────┤
│  [全部] [中餐] [西餐] [日料]      │
│  [韩餐] [东南亚] [快餐] [自定义]  │
├──────────────────────────────────┤
│  食物列表：                      │
│  ┌────────────────────────────┐  │
│  │ 🍜 红烧牛肉面  [中餐]      │  │
│  ├────────────────────────────┤  │
│  │ 🍕 披萨        [西餐]      │  │
│  ├────────────────────────────┤  │
│  │ 🍣 寿司        [日料]      │  │
│  ├────────────────────────────┤  │
│  │ 🍔 自定义汉堡   [自定义] [✏️][🗑️] │
│  └────────────────────────────┘  │
│         [+ 添加食物]             │
└──────────────────────────────────┘
```

#### 两种随机方式

##### 1. Canvas 转盘旋转（"转一转"按钮）

- 使用 Canvas 2D 绘制彩色转盘，每个扇区对应一道食物
- 随机选择目标扇区，旋转 5-7 圈，动画持续 4-5 秒
- 使用三次方缓动（ease-out）实现自然减速效果
- 旋转结束后根据指针指向的扇区确定结果
- 触发重度震动反馈

##### 2. 老虎机轮播（"随机"按钮）

- 25-34 步快速轮播，延迟时间随步数递增实现减速
- 最终停留在随机选中的食物上
- 触发震动反馈

#### 自定义食物管理

- **添加**: 点击"添加食物"弹窗，输入名称（自动归属"自定义"分类），校验非空和去重
- **编辑**: 点击编辑按钮修改自定义食物名称
- **删除**: 点击删除按钮，确认后删除（仅限自定义食物，预设食物不可删除）
- **持久化**: 自定义食物保存到 localStorage (`eat_what_custom_foods`)

#### 数据规模

- 预设 54 种食物，分 7 类：中餐(20) / 西餐(8) / 日料(7) / 韩餐(6) / 东南亚(5) / 快餐(8)
- 支持按分类筛选，转盘自动更新对应食物

## 🔧 核心工具模块

### 1. 推荐引擎（utils/recipe-engine.js）

应用的"大脑"，负责所有菜谱推荐逻辑：

| 函数 | 功能 |
|---|---|
| `getHotRecipes()` | 从热门菜谱中随机取 4 道 |
| `getRecipesByCategory()` | 按分类筛选 + 打分排序 |
| `searchRecipes()` | 关键词搜索（名称/标签/食材） |
| `getDietRecipes()` | 获取减脂友好菜谱 |
| `getRecipeDetail()` | 精确/模糊查找单道菜谱 |
| `matchByIngredients()` | 冰箱食材匹配率计算 |
| `aiRecommendByIngredients()` | 讯飞星火 AI 推荐 fallback |

**打分规则**: 分类匹配+30、减脂标记+20、名称关键词+25、标签匹配+15、食材匹配+15/个、难度适中+5、随机因子0-20

### 2. 投票系统（utils/cloud.js）

纯本地存储的投票方案：

- 房间数据存储在 `wx.getStorageSync('vote_rooms')`
- 历史记录最多保留 50 条
- 房间码：6 位大写字母+数字（排除易混淆字符）
- 投票码格式：`投票码:XXXXXX|encoded_data`

### 3. 数据文件

| 文件 | 内容 |
|---|---|
| `data/recipes.js` | 80+ 道预置菜谱（中餐35/西餐15/日料10/韩餐8/减脂15） |
| `data/videos.js` | Bilibili 视频链接映射（73/74 道菜有视频） |
| `utils/foods.js` | 54 种预设食物（转盘用） |
| `utils/icons.js` | 食材/菜谱图标映射 |
